# Project Log

Append-only chronological record of milestones across this project's life. **Never edited, never pruned.** Survives every ship. The durable narrative.

## How to read

Each line is a single self-contained record:

```
<ISO-8601 timestamp>  <stage>  <kind>  <one-line description>  [→ <pointer>]
```

`<kind>` is one of: `started`, `shipped`, `decision`, `blocker`, `unblock`, `replan`, `parked`, `note`. The optional pointer is a checkpoint tag, commit sha, branch name, or progress-file path.

## How to write

- One milestone, one line. If it doesn't fit on one line, it's two milestones or a progress-file note.
- Append at the moment the milestone happens — not retrospectively, not in a batch at session end.
- Never edit existing entries. Mistakes get a follow-up `note` line, not a rewrite.
- The `started` and `shipped` kinds are auto-fired by the skill (`/log` and `/ship`); decisions, blockers, unblocks, parks, and notes are written by the coding agent (or the user via `/log`) at the moment they occur.

## How it relates to other artifacts

- **PRD** (`docs/PRD.md`) — vision; rarely changes.
- **Completion plan** (`docs/local/completion-plan-<date>.md`) — ordering of stages; revised on `/replan`.
- **This file** — append-only milestone history; the single durable narrative.
- **Per-effort progress** (`docs/local/progress/*.md`) — sprint-scoped detail; distilled into one line here on ship, then deleted.
- **STATE.md** (`docs/local/STATE.md`) — derived "right now" snapshot; regenerated by `/where` from this file's tail plus the active progress file's tail.

`/where` and `/roadmap` read this file. `/where` regenerates STATE.md from its tail. `/roadmap` summarizes the full file when small; once the log crosses **500 lines OR 6 months**, `/roadmap` switches to roll-up mode and compresses older entries to one line per stage (the file itself stays append-only forever — the roll-up is just a render).
